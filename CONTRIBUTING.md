# Contributing to FixDiscordEgypt

Thank you for your interest in helping others access Discord in Egypt! This guide explains how to contribute.

## Ways to Contribute

### 1. Report a New Method

Found a way to access Discord that isn't listed?

1. Fork this repository
2. Add your method to `README.md` following the existing format
3. Include: description, pros, cons, requirements, steps, and download links
4. Submit a Pull Request

### 2. Report Issues

Something not working?

1. Go to [Issues](../../issues)
2. Click "New Issue"
3. Select the appropriate template
4. Fill in the details

### 3. Fix Errors

Found a typo or outdated information?

1. Fork this repository
2. Make your changes
3. Submit a Pull Request

### 4. Add Translations

Want to translate the guide?

1. Copy `README.md` to `README_XX.md` (XX = language code)
2. Translate the content
3. Add a language link to the main README
4. Submit a Pull Request

---

## How to Submit a Pull Request

### Step 1: Fork

Click the "Fork" button at the top right of this repository.

### Step 2: Clone

```bash
git clone https://github.com/YOUR-USERNAME/FixDiscordEgypt.git
cd FixDiscordEgypt
```

### Step 3: Create a Branch

```bash
git checkout -b your-feature-name
```

### Step 4: Make Changes

Edit the files as needed.

### Step 5: Commit

```bash
git add .
git commit -m "Add: brief description of your change"
```

### Step 6: Push

```bash
git push origin your-feature-name
```

### Step 7: Open PR

Go to your fork on GitHub and click "Compare & pull request".

---

## Commit Message Format

Use clear, descriptive commit messages:

| Type | Example |
|------|---------|
| Add new content | `Add: Mullvad VPN method` |
| Fix errors | `Fix: typo in WARP section` |
| Update content | `Update: Vesktop download link` |
| Remove content | `Remove: outdated method` |
| Translate | `Translate: add French version` |

---

## Method Template

When adding a new method, use this structure:

```markdown
## Method X: [Name]

### Description
[What it is and how it works]

### Pros
- [Advantage 1]
- [Advantage 2]

### Cons
- [Limitation 1]
- [Limitation 2]

### Requirements
| Requirement | Details |
|-------------|---------|
| **OS** | [Supported systems] |
| **Account** | [Required or not] |

### Installation Steps
1. [Step 1]
2. [Step 2]
3. [Step 3]

### Official Download Links
| Platform | Link |
|----------|------|
| [Platform] | [Full URL] |

### Troubleshooting
| Issue | Solution |
|-------|----------|
| [Problem] | [Fix] |
```

---

## Guidelines

- **Test your method** — Only submit methods you've verified work in Egypt
- **Use official links** — No shortened URLs or unofficial mirrors
- **Keep it simple** — Write for beginners
- **Be accurate** — Don't exaggerate pros or hide cons
- **Respect privacy** — Don't recommend tools that harvest user data
